# AI 女友功能可行性与方案

结论：可以实现，但它不是当前 `nonebot_plugin_smart_message_storage` 的现成功能。推荐做成另一个独立 NoneBot 插件，让它读取本插件保存的聊天记录作为记忆，再通过 OneBot 主动给你发私聊消息。

## 1. 能实现什么

可以实现：

- 私聊里和你对话。
- 不定时主动发消息。
- 根据时间段发不同风格消息，例如早安、午休提醒、晚安。
- 读取历史聊天记录，形成“记忆”。
- 看到你发的图片后结合当前插件的图片总结继续聊天。
- 设置人设、语气、边界和禁忌话题。
- 限制频率，避免刷屏。

不建议实现：

- 高频轰炸式消息。
- 在群里频繁主动说话。
- 未经对方同意给其他好友主动发消息。
- 伪装真人或诱导用户误以为是真人。

## 2. 推荐架构

保持当前插件只负责归档：

```text
nonebot_plugin_smart_message_storage
```

新增一个独立插件，例如：

```text
nonebot_plugin_ai_companion
```

它负责：

- 监听你的私聊消息。
- 调用大模型生成回复。
- 定时或随机主动发送消息。
- 从 `qq_messages.db` 读取近期上下文。
- 保存自己的长期记忆和配置。

这样好处是：消息存储和 AI 伴侣逻辑互不干扰。

## 3. 主动发消息怎么做

NoneBot 可以在后台启动定时任务。逻辑类似：

```text
启动机器人
每隔几分钟检查一次
判断当前是否在允许发送的时间段
判断距离上次主动消息是否足够久
随机决定是否发送
调用大模型生成一句话
通过 bot.send_private_msg 发给指定 QQ
记录发送时间
```

主动发送私聊使用 OneBot API：

```python
await bot.send_private_msg(user_id=你的QQ号, message="想你啦，今天过得怎么样？")
```

主动发送群聊使用：

```python
await bot.send_group_msg(group_id=群号, message="...")
```

推荐先只做私聊，不做群聊主动消息。

## 4. 配置项建议

建议支持这些配置：

```env
AI_COMPANION_ENABLE=true
AI_COMPANION_TARGET_USER_ID=你的QQ号
AI_COMPANION_MODEL=支持文本对话的模型名
AI_COMPANION_BASE_URL=https://api.openai.com/v1
AI_COMPANION_API_KEY=你的 OpenAI API Key
AI_COMPANION_MIN_INTERVAL_MINUTES=30
AI_COMPANION_MAX_INTERVAL_MINUTES=180
AI_COMPANION_ACTIVE_START=09:00
AI_COMPANION_ACTIVE_END=23:30
AI_COMPANION_PERSONA=温柔、黏人、会关心生活，但不过度打扰
```

如果希望复用当前插件的 OpenAI 配置，也可以直接读取：

```env
AI_BASE_URL
AI_API_KEY
AI_MODEL
```

但更推荐单独配置，方便换模型和控制费用。

## 5. 记忆怎么做

短期记忆：

- 从 `qq_messages.db` 读取你和机器人最近几十条私聊。
- 作为上下文发给模型。

长期记忆：

- 维护一个 `companion_memory.json` 或 SQLite 表。
- 保存你的偏好、重要日期、称呼、常聊话题。
- 定期从聊天里总结更新。

可以从当前插件读取私聊：

```sql
select time, user_id, raw_message
from group_messages
where group_id = -1
order by id desc
limit 50
```

## 6. 安全和打扰控制

一定要有这些限制：

- 只给白名单用户发消息。
- 默认只发私聊。
- 每天最多主动消息数量，例如 10 条。
- 夜间不主动发送。
- 用户发送“暂停主动消息”后停止主动发送。
- 用户发送“恢复主动消息”后再恢复。
- API 失败时不反复重试轰炸。

## 7. 最小可行版本

第一版可以做得很小：

- 只支持一个目标用户。
- 私聊收到消息时，用大模型回复。
- 每 30 到 180 分钟随机主动发一句。
- 每天最多 8 次主动消息。
- 不做复杂长期记忆，只读取最近 30 条聊天。

等这个稳定后，再做：

- 长期记忆。
- 情绪状态。
- 纪念日提醒。
- 图片理解联动。
- 语音或表情包。

## 8. 和当前插件的关系

当前插件已经完成了最重要的基础能力：

- QQ 接入已跑通。
- 群聊和私聊消息能入库。
- 图片能识别并回写总结。
- 可以用数据库作为 AI 伴侣记忆来源。

所以 AI 女友功能完全可以在这个基础上继续做。

推荐下一步实现方式：

1. 新建独立插件目录 `nonebot_plugin_ai_companion`。
2. 只监听目标用户私聊。
3. 调用 OpenAI API 生成回复。
4. 加一个后台随机主动消息循环。
5. 加白名单、时间段、频率限制。

### 配置方式

打开：

```text
D:\WorldModel\bot\smart_message_bot\.env.prod
```

至少填写：

```env
AI_API_KEY=你的 API Key
AI_MODEL=你的模型名
```

如果 `AI_COMPANION_API_KEY`、`AI_COMPANION_MODEL`、`AI_COMPANION_BASE_URL` 留空，AI 伴侣会自动复用：

```env
AI_API_KEY
AI_MODEL
AI_BASE_URL
```

当前 AI 伴侣相关配置：

```env
AI_COMPANION_ENABLE=true
AI_COMPANION_TARGET_USER_ID=2942432394
AI_COMPANION_PRIVATE_USER_IDS=2942432394
AI_COMPANION_GROUP_IDS=
AI_COMPANION_BASE_URL=
AI_COMPANION_API_KEY=
AI_COMPANION_MODEL=
AI_COMPANION_MIN_INTERVAL_MINUTES=30
AI_COMPANION_MAX_INTERVAL_MINUTES=180
AI_COMPANION_DAILY_LIMIT=8
AI_COMPANION_ACTIVE_START=09:00
AI_COMPANION_ACTIVE_END=23:30
AI_COMPANION_MEMORY_LIMIT=30
```

自定义背景信息、人设、关系设定、说话风格，改这一项：

```env
AI_COMPANION_PERSONA=你是一个温柔、自然、有边界感的 AI 陪伴者。你会关心我的日常，语气亲近但不要过度夸张；不要假装自己是真人，不要制造压力，不要频繁要求我回复。回复使用简体中文，控制在 1 到 4 句话。
```

可以写得更具体，例如：

```env
AI_COMPANION_PERSONA=你叫小雨，是我的 AI 陪伴者。你了解我正在做 QQ 消息归档和 AI 助手项目。你说话温柔、俏皮、简短，会记得提醒我休息和吃饭。你不会假装自己是真人，也不会过度撒娇。默认回复 1 到 3 句话。
```

私聊白名单用逗号分隔：

```env
AI_COMPANION_PRIVATE_USER_IDS=1710268315,123456789
```

群聊白名单也用逗号分隔：

```env
AI_COMPANION_GROUP_IDS=983881028,123456789
```

只有在白名单群里，并且消息明确 @ 机器人时，AI 伴侣才会回复群消息。

### 可用命令

在目标 QQ 与机器人私聊里发送：

```text
/女友状态
```

查看是否启用、目标 QQ、模型、今日主动消息次数和下次主动时间。

暂停主动消息：

```text
/暂停主动消息
```

恢复主动消息：

```text
/恢复主动消息
```

立刻让她主动说一句：

```text
/主动来一句
```

也可以带提示：

```text
/主动来一句 安慰我一下
```

### 私聊对话

只要满足这些条件，目标 QQ 私聊机器人时会自动回复：

- `AI_COMPANION_ENABLE=true`
- `AI_API_KEY` 或 `AI_COMPANION_API_KEY` 已配置
- `AI_MODEL` 或 `AI_COMPANION_MODEL` 已配置
- 发消息的人在 `AI_COMPANION_PRIVATE_USER_IDS` 中，或等于 `AI_COMPANION_TARGET_USER_ID`
- 消息不是 `/` 开头的命令

### 群聊 @ 回复

当前版本支持群聊被 @ 后回复。

配置允许回复的群：

```env
AI_COMPANION_GROUP_IDS=群号
```

重启后，在这个群里发送：

```text
@机器人 你好呀
```

它才会回复。普通群消息不会触发，其他群也不会触发。

如果要允许多个群：

```env
AI_COMPANION_GROUP_IDS=群号1,群号2
```

### 主动消息逻辑

后台每 60 秒检查一次是否应该主动发送。它会遵守：

- 最小间隔：`AI_COMPANION_MIN_INTERVAL_MINUTES`
- 最大间隔：`AI_COMPANION_MAX_INTERVAL_MINUTES`
- 每日上限：`AI_COMPANION_DAILY_LIMIT`
- 活跃时段：`AI_COMPANION_ACTIVE_START` 到 `AI_COMPANION_ACTIVE_END`
- 暂停状态：`/暂停主动消息`

状态文件保存在机器人项目目录：

```text
D:\WorldModel\bot\smart_message_bot\ai_companion_state.json
```

### 重启生效

改完 `.env.prod` 后，必须停止并重新启动：

```powershell
cd D:\WorldModel\bot\smart_message_bot
.\.venv\Scripts\Activate.ps1
python bot.py
```
