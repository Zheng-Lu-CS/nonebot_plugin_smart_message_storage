# 故障排查

每个问题都按“现象、原因、检查命令、修复步骤”来处理。

## nb 无法识别

现象：

```text
nb : 无法将“nb”项识别为 cmdlet、函数、脚本文件或可运行程序的名称
```

原因：

`nb` 是 `nb-cli` 提供的命令，不是 Python 自带命令。

检查：

```powershell
python -m pip show nb-cli
```

修复：

本教程不需要 `nb`。继续使用：

```powershell
python bot.py
```

如果确实要安装：

```powershell
python -m pip install nb-cli
```

## pdm 无法识别

现象：

```text
pdm : 无法将“pdm”项识别为 cmdlet、函数、脚本文件或可运行程序的名称
```

原因：

PDM 没有安装或不在 PATH。

检查：

```powershell
python -m pip show pdm
```

修复：

本教程不需要 PDM。使用 `venv + pip` 即可。

## PowerShell 无法激活虚拟环境

现象：

```text
无法加载文件 .venv\Scripts\Activate.ps1，因为在此系统上禁止运行脚本
```

原因：

PowerShell 执行策略限制脚本运行。

检查：

```powershell
Get-ExecutionPolicy
```

修复：

只在当前窗口放开：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

## 依赖安装失败

现象：

`pip install` 报网络错误、找不到包、构建失败。

原因：

可能是网络、pip 太旧、Python 版本过新或包构建依赖问题。

检查：

```powershell
python --version
python -m pip --version
python -m pip debug --verbose
```

修复：

先升级基础工具：

```powershell
python -m pip install -U pip setuptools wheel
```

确认使用 Python 3.10：

```powershell
py -3.10 -m venv .venv
.\.venv\Scripts\Activate.ps1
python --version
```

然后重新安装：

```powershell
python -m pip install "nonebot2[fastapi]" nonebot-adapter-onebot nonebot-plugin-localstore
python -m pip install -e D:\WorldModel\project\nonebot_plugin_smart_message_storage
```

如果日志里出现 `ProxyError`、`FileNotFoundError`、`SSLEOFError`，并且系统代理类似 `127.0.0.1:7897`，可以只在当前命令里绕过系统代理：

```powershell
$env:NO_PROXY='*'
$env:no_proxy='*'
python -m pip install fastapi uvicorn -i https://pypi.org/simple --trusted-host pypi.org --trusted-host files.pythonhosted.org
```

也可以先检查系统代理：

```powershell
netsh winhttp show proxy
Get-ItemProperty 'HKCU:\Software\Microsoft\Windows\CurrentVersion\Internet Settings' | Select-Object ProxyEnable,ProxyServer,AutoConfigURL
```

## Python 3.13 兼容性风险

现象：

依赖安装报错，或运行时出现奇怪兼容问题。

原因：

仓库要求 Python 3.10+，但机器人生态通常在 3.10 到 3.12 更稳。当前机器默认 `python` 是 3.13，不建议用于这个项目。

检查：

```powershell
python --version
py -0p
```

修复：

明确使用 Python 3.10：

```powershell
py -3.10 -m venv .venv
.\.venv\Scripts\Activate.ps1
python --version
```

## OneBot 连不上 NoneBot

现象：

NapCat 显示连接失败，NoneBot 没有收到连接日志。

原因：

可能是 NoneBot 没启动、端口不对、WebSocket 路径不对、防火墙拦截，或 NapCat 没启用反向 WebSocket。

检查：

```powershell
netstat -ano | findstr :8080
```

确认 `.env.prod`：

```env
DRIVER=~fastapi
HOST=127.0.0.1
PORT=8080
```

修复：

1. 先启动 NoneBot：`python bot.py`。
2. NapCat 反向 WebSocket 地址填：

```text
ws://127.0.0.1:8080/onebot/v11/ws
```

3. 如果 NapCat 和 NoneBot 不在同一台机器，不能用 `127.0.0.1`，要改成 NoneBot 所在机器的局域网 IP，并检查防火墙。

## NapCat WebSocket 连接返回 404

现象：

NapCat 日志显示：

```text
Unexpected server response: 404
```

NoneBot / Uvicorn 日志显示：

```text
Unsupported upgrade request.
No supported WebSocket library detected.
Please use "pip install 'uvicorn[standard]'", or install 'websockets' or 'wsproto' manually.
GET /onebot/v11/ws HTTP/1.1 404
```

原因：

Uvicorn 已经启动 HTTP 服务，但当前虚拟环境缺少 WebSocket 支持库。NapCat 发来的 WebSocket 握手没有被正确升级，最终被当成普通 HTTP 请求返回 404。

检查：

```powershell
python -m pip show websockets wsproto uvicorn
```

修复：

```powershell
cd D:\WorldModel\bot\smart_message_bot
.\.venv\Scripts\Activate.ps1
python -m pip install websockets
```

如果 pip 被系统代理影响，可以临时绕过代理：

```powershell
$env:NO_PROXY='*'
$env:no_proxy='*'
python -m pip install websockets -i https://pypi.org/simple --trusted-host pypi.org --trusted-host files.pythonhosted.org
```

安装后关闭所有 `bot.py`，只用虚拟环境重新启动：

```powershell
python bot.py
```

## 数据库为空

现象：

`qq_messages.db` 存在，但 `group_messages` 没有消息。

原因：

机器人没有成功接入 QQ，或小号没有收到目标会话里的新消息。

检查：

```powershell
python -c "import sqlite3; con=sqlite3.connect('qq_messages.db'); print(con.execute('select count(*) from group_messages').fetchone())"
```

修复：

- 确认 NapCat 在线。
- 确认小号在目标群里。
- 在群里发送一条新消息。
- 看 NoneBot 日志是否出现消息处理记录。

## /查消息 返回查询失败

现象：

QQ 里发送 `/查消息 关键词` 后机器人回复“查询失败”，或日志出现字体相关错误。

原因：

源码使用 `ImageFont.truetype("simkai.ttf", size=20)`，运行目录找不到 `simkai.ttf`。

检查：

```powershell
Set-Location D:\WorldModel\bot\smart_message_bot
Test-Path .\simkai.ttf
Test-Path C:\Windows\Fonts\simkai.ttf
```

修复：

```powershell
Copy-Item C:\Windows\Fonts\simkai.ttf D:\WorldModel\bot\smart_message_bot\simkai.ttf
```

重启机器人后再试 `/查消息`。

## 图片下载失败

现象：

日志出现：

```text
[AI识图] 下载或缓存图片失败，保留原始 CQ 码
```

原因：

OneBot 上报的图片 URL 不可访问、图片过期、本地路径不存在，或网络请求失败。

检查：

- 看日志中的 `message_id` 和错误。
- 重新发送一张图片测试。
- 确认 NapCat 能正常获取图片。

修复：

- 重新发送原图。
- 确认机器人和 NapCat 在同一环境下能访问图片 URL。
- 不要等太久再识别过期图片。

## OpenAI 401 / 403

现象：

日志显示 AI 请求失败，HTTP 401 或 403。

原因：

API Key 错误、过期、无权限、账户不可用，或环境变量没有被正确读取。

检查：

```powershell
Get-Content .\.env.prod
```

不要把完整 Key 发给别人排查。

修复：

- 重新生成 API Key。
- 确认 `.env.prod` 中 `AI_API_KEY` 不为空。
- 重启机器人。

## 模型不支持图片

现象：

日志显示模型不支持图片、内容类型错误，或 AI 返回空内容。

原因：

`AI_MODEL` 不是视觉模型，或不支持当前接口形态。

检查：

查看 OpenAI 视觉文档：

```text
https://platform.openai.com/docs/guides/images-vision
```

修复：

换成支持图片输入、Chat Completions、JSON 输出的模型，然后重启机器人。

## AI 返回非 JSON

现象：

日志显示：

```text
AI returned invalid JSON
```

或：

```text
AI model returned an image link; it looks like a generation model, not a vision model
```

原因：

模型没有按插件要求输出 JSON，或者你填成了图像生成模型。

检查：

`.env.prod` 中的 `AI_MODEL`。

修复：

- 换成视觉理解模型，不要用图像生成模型。
- 确认模型支持 JSON 输出。

## /识别 提示数据库找不到回复消息

现象：

回复一条图片消息发送 `/识别`，机器人提示：

```text
没有在数据库中找到回复的消息。
```

原因：

被回复的图片消息发生在机器人上线前，或当时插件没有成功入库。

检查：

```powershell
python -c "import sqlite3; mid=123456; con=sqlite3.connect('qq_messages.db'); print(con.execute('select * from group_messages where message_id=?',(mid,)).fetchall())"
```

修复：

重新发送这张图片，让机器人先记录它，再回复 `/识别`。

## /立即识别 返回 0 张

现象：

机器人回复：

```text
已提交当前会话待识别图片 0 张。
```

原因：

当前会话没有 pending 图片任务，或没有配置 `AI_API_KEY`，或图片消息没有成功缓存。

检查：

- 是否已经配置 `AI_API_KEY`。
- 是否在当前群/私聊发送过图片。
- 启动日志是否提示 AI image recognition is disabled。

修复：

1. 配好 `AI_API_KEY` 和 `AI_MODEL`。
2. 重启机器人。
3. 重新发送图片。
4. 再执行 `/立即识别`。

## 导出文件为空

现象：

CSV/JSON 生成了，但没有数据。

原因：

目标群号或好友 QQ 写错；私聊查询条件不对；消息还没入库。

检查：

查看数据库有哪些 group_id：

```powershell
python -c "import sqlite3; con=sqlite3.connect('qq_messages.db'); [print(r) for r in con.execute('select group_id,user_id,count(*) from group_messages group by group_id,user_id order by count(*) desc')]"
```

修复：

- 群聊导出用真实群号：`group_id=<群号>`。
- 私聊导出用：`group_id=-1 AND user_id=<好友QQ>`。
- 如果小号不是和这个好友直接私聊，数据库里不会出现这条私聊会话。
