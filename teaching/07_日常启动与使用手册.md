# 日常启动与使用手册

这份手册用于以后每天实际使用，不再重复安装步骤。

## 1. 每次启动顺序

推荐顺序：

1. 启动 NapCat。
2. 启动 NoneBot。
3. 确认 OneBot 连接成功。
4. 在 QQ 里发送测试消息。

## 2. 启动 NapCat

打开：

```text
C:\Users\华为\Downloads\NapCat.Shell.Windows.OneKey\NapCat.44498.Shell
```

双击：

```text
napcat.bat
```

如果需要进入 WebUI：

```text
http://127.0.0.1:6099/webui
```

登录密钥在：

```text
C:\Users\华为\Downloads\NapCat.Shell.Windows.OneKey\NapCat.44498.Shell\versions\9.9.26-44498\resources\app\napcat\config\webui.json
```

不要把 WebUI 登录密钥发给别人。

## 3. 启动 NoneBot

新开一个 PowerShell：

```powershell
cd D:\WorldModel\bot\smart_message_bot
.\.venv\Scripts\Activate.ps1
python bot.py
```

看到下面这类日志说明服务启动成功：

```text
Running NoneBot...
Uvicorn running on http://127.0.0.1:8080
```

看到下面这类日志说明 NapCat 已连接：

```text
WebSocket /onebot/v11/ws [accepted]
OneBot V11 | Bot 2942432394 connected
connection open
```

如果看到 `No supported WebSocket library detected` 或 `404`，看 `06_故障排查.md` 的 WebSocket 章节。

## 4. 配置 OpenAI API Key

打开：

```text
D:\WorldModel\bot\smart_message_bot\.env.prod
```

填写：

```env
AI_BASE_URL=https://api.openai.com/v1
AI_API_KEY=你的 OpenAI API Key
AI_MODEL=支持视觉输入的模型名
```

注意：

- API Key 不要发给别人。
- 改完 `.env.prod` 后必须重启 `python bot.py`。
- 当前插件使用 OpenAI 兼容的 `chat/completions` 接口，模型要支持图片输入和 JSON 输出。

## 5. 平时怎么用

机器人启动并连接 NapCat 后，插件会自动存消息。你不需要手动输入“开始记录”。

群聊记录：

- 小号在群里。
- NoneBot 和 NapCat 都在线。
- 群里新消息会自动写入 `qq_messages.db`。

私聊记录：

- 对方和小号私聊。
- 新消息会自动写入 `group_id=-1` 的私聊记录。

图片识别：

- 配好 `AI_API_KEY` 后，图片消息会进入待识别队列。
- 默认攒够 5 张、超时、或手动命令触发时提交识别。

## 6. QQ 内命令

查当前群或当前私聊：

```text
/查消息 关键词
```

查指定群：

```text
/查消息 群号 关键词
```

立即识别当前会话待处理图片：

```text
/立即识别
```

识别所有会话待处理图片，只有 `SUPERUSERS` 可用：

```text
/立即识别 全部
```

回复一条图片消息后识别这张图：

```text
/识别
```

## 7. 检查数据库

查看最新 5 条消息：

```powershell
cd D:\WorldModel\bot\smart_message_bot
.\.venv\Scripts\Activate.ps1
python -c "import sqlite3; con=sqlite3.connect('qq_messages.db'); print(con.execute('select id,time,group_id,user_id,raw_message from group_messages order by id desc limit 5').fetchall())"
```

看总条数：

```powershell
python -c "import sqlite3; con=sqlite3.connect('qq_messages.db'); print(con.execute('select count(*) from group_messages').fetchone())"
```

## 8. 停止服务

停止 NoneBot：

- 在运行 `python bot.py` 的 PowerShell 里按 `Ctrl+C`。

停止 NapCat：

- 关闭 NapCat 的 cmd 窗口，或运行安装目录里的 `napcat.kill.qq.bat`。

## 9. 常见日常问题

如果数据库没有新消息：

- 确认 NapCat 仍在线。
- 确认 NoneBot 窗口仍开着。
- 确认日志有 `Bot ... connected`。
- 确认没有又用全局 Python 启动了 `bot.py`。

检查当前是谁占用 8080：

```powershell
netstat -ano | findstr :8080
```

检查正在运行的 `bot.py`：

```powershell
Get-CimInstance Win32_Process | Where-Object { $_.CommandLine -like '*bot.py*' } | Select-Object ProcessId,Name,CommandLine
```

正确的命令行应包含：

```text
D:\WorldModel\bot\smart_message_bot\.venv\Scripts\python.exe
```
