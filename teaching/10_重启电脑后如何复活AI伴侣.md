# 重启电脑后如何复活 AI 伴侣

这份文档给你以后每天使用。目标是：电脑重启后，让 QQ 机器人、消息存储、图片识别、AI 伴侣重新工作。

## 1. 总体顺序

每次重启电脑后，按这个顺序来：

1. 启动 NapCat，并确认机器人 QQ 已登录。
2. 启动 NoneBot 项目。
3. 确认 NapCat 已连接到 NoneBot。
4. 在 QQ 私聊或群聊里测试 AI 伴侣。

不要同时启动多个 `bot.py`。只能保留一个，并且必须是虚拟环境里的 Python。


双击运行：

```text
napcat.bat
```

如果弹出二维码，使用机器人 QQ 号扫码登录。

当前机器人 QQ 是：

```text
2942432394
```

登录后，NapCat 控制台里应该能看到 QQ 消息收发日志。

## 3. 检查 NapCat 网络配置

NapCat WebUI 地址一般是：

```text
http://127.0.0.1:6099/webui
```

进入后看：

```text
网络配置 -> Websocket客户端
```

需要有一个启用的 WebSocket 客户端配置，URL 应该是：

```text
ws://127.0.0.1:8080/onebot/v11/ws
```

注意：

- 类型是 `Websocket客户端`。
- 必须启用。
- URL 端口必须是 `8080`。
- 路径必须是 `/onebot/v11/ws`。

## 4. 启动 NoneBot

打开 PowerShell，严格执行：

```powershell

D:\WorldModel\bot\smart_message_bot\.venv\Scripts\python.exe bot.py
```

推荐直接用完整路径启动，不要点 VSCode 右上角运行按钮，避免误用全局 Python。

启动成功后，日志里应该看到类似：

```text

Loaded plugin "plugins.ai_companion"
Uvicorn running on http://127.0.0.1:8080
OneBot V11 | Bot 2942432394 connected
```

如果看到：

```text

```

就说明 NapCat 和 NoneBot 连上了。

## 5. 防止重复启动

如果行为很怪，比如私聊能回、群聊不回，或者日志重复刷，很可能是多个 `bot.py` 同时在跑。

检查命令：

```powershell
Get-CimInstance Win32_Process | Where-Object { $_.CommandLine -like '*bot.py*' } | Select-Object ProcessId,Name,CommandLine
```

正常情况下，只应该看到一个：

```text
D:\WorldModel\bot\smart_message_bot\.venv\Scripts\python.exe bot.py
```

如果看到这个，说明启动错了：

```text
D:\Python310\python.exe bot.py
```

可以停掉所有 `bot.py`，再重新用虚拟环境启动：

```powershell
Get-CimInstance Win32_Process | Where-Object { $_.CommandLine -like '*bot.py*' } | ForEach-Object { Stop-Process -Id $_.ProcessId -ErrorAction SilentlyContinue }

cd D:\WorldModel\bot\smart_message_bot
D:\WorldModel\bot\smart_message_bot\.venv\Scripts\python.exe bot.py
```

## 6. 日常怎么和她聊天

### 私聊

直接用你的 QQ 私聊机器人 QQ：

```text
你好呀
```

只要你的 QQ 在私聊白名单里，她就会自动回复。

当前私聊白名单配置在：

```text
D:\WorldModel\bot\smart_message_bot\.env.prod
```

对应配置：

```env
AI_COMPANION_PRIVATE_USER_IDS=1710268315
```

多个 QQ 用英文逗号分隔：

```env
AI_COMPANION_PRIVATE_USER_IDS=1710268315,123456789
```

### 群聊

群聊不会回复所有消息，只会在指定群里被触发时回复。

当前群聊白名单：

```env
AI_COMPANION_GROUP_IDS=983881028
```

群里可以这样触发：

```text
@苏语桐 在吗
```

或者真正艾特机器人：

```text
@机器人 在吗
```

触发名配置：

```env
AI_COMPANION_TRIGGER_NAMES=苏语桐
```

如果想改名字，例如改成“小桐”：

```env
AI_COMPANION_TRIGGER_NAMES=小桐
```

多个名字：

```env
AI_COMPANION_TRIGGER_NAMES=苏语桐,小桐,语桐
```

注意：这只是文本触发名。如果想让 QQ 真正 @ 时显示“苏语桐”，需要在 QQ 群里把机器人账号的群昵称改成“苏语桐”。

## 8. 主动消息怎么生效

主动消息由这些配置控制：

```env
AI_COMPANION_MIN_INTERVAL_MINUTES=30
AI_COMPANION_MAX_INTERVAL_MINUTES=180
AI_COMPANION_DAILY_LIMIT=8
AI_COMPANION_ACTIVE_START=09:00
AI_COMPANION_ACTIVE_END=23:30
```

含义：

- 最短 30 分钟主动一次。
- 最长 180 分钟主动一次。
- 每天最多主动 8 次。
- 只在 09:00 到 23:30 之间主动发消息。

如果你刚重启，主动消息不会马上来。可以先用：

```text
/主动来一句
```

测试她是否正常。

## 9. 回复延迟

她收到消息后不会必须立刻回复，可以设置随机延迟：

```env
AI_COMPANION_REPLY_DELAY_MIN_SECONDS=1
AI_COMPANION_REPLY_DELAY_MAX_SECONDS=4
```

想马上回复：

```env
AI_COMPANION_REPLY_DELAY_MIN_SECONDS=0
AI_COMPANION_REPLY_DELAY_MAX_SECONDS=0
```

想更像真人：

```env
AI_COMPANION_REPLY_DELAY_MIN_SECONDS=3
AI_COMPANION_REPLY_DELAY_MAX_SECONDS=10
```

## 10. 自定义人设

打开：

```text
D:\WorldModel\bot\smart_message_bot\.env.prod
```

修改：

```env
AI_COMPANION_PERSONA="你的名字是“苏语桐”，你说话温柔、自然、简短，像 QQ 私聊。不要输出 [表情]、[图片]、[CQ:...] 这类占位符。"
```

你可以写：

- 她叫什么。
- 她和你的关系。
- 她了解你的哪些背景。
- 她应该怎么称呼你。
- 回复长短。
- 禁止事项。

改完 `.env.prod` 后，必须重启 NoneBot 才生效。

## 11. 图片和表情

图片输入配置：

```env
AI_COMPANION_ENABLE_IMAGE_INPUT=true
AI_COMPANION_MAX_IMAGES_PER_MESSAGE=3
```

如果模型和 API 服务商支持视觉输入，她会尽量理解你发的图片。

普通 QQ 表情和表情包，目前会作为情绪提示给模型，不保证能准确识别表情包内容。

## 14. 如果不回复

先看日志里有没有：

```text
[AI伴侣] 跳过群聊
```

常见原因：

- 群号不在 `AI_COMPANION_GROUP_IDS`。
- 没有写 `@苏语桐`，也没有真正 @ 机器人。
- 同时跑了多个 `bot.py`。
- API Key 失效或额度不足。
- NapCat 没连上 NoneBot。

最常用修复：

```powershell
Get-CimInstance Win32_Process | Where-Object { $_.CommandLine -like '*bot.py*' } | ForEach-Object { Stop-Process -Id $_.ProcessId -ErrorAction SilentlyContinue }

cd D:\WorldModel\bot\smart_message_bot
D:\WorldModel\bot\smart_message_bot\.venv\Scripts\python.exe bot.py
```

然后重新发：

```text
/女友状态
```

和：

```text
@苏语桐 在吗
```
